# Huntd Web Platform — QA Case Study

**Role:** QA Engineer 
**Scope:** Web functional testing, requirements decomposition, RTM, permission matrix, TestRail execution, defect reporting.

---

## TL;DR
- **Full requirements decomposition mapped to Jira stories and RTM**  
- **147 executed test cases** → **131 passed**, **17 failed**  
- **Deliverables included:** Test Plan, RTM, Permission Table, Test Cases (TestRail export), Screenshots (stories, bugs, metrics)

## 🔍 Key Findings
- All critical user flows were covered (authentication, profile, candidates, jobs, chats).  
- Several functional and UI defects were identified and documented with evidence.  
- Full traceability achieved: requirements → RTM → test cases → execution results.  
- Testing revealed inconsistencies in filters, chat behavior.
